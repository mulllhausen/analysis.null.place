Have you ever looked at a large route table and thought, "Wow! There are just too many rules to see what is going on"? With large route tables even the most experienced sysadmin could be forgiven for not knowing which gateway a destination IP will use. This is because networks can legally overlap, and when they do then it is the applicable rule with the highest mask which takes precedence - but that is hard to see from a big list of IP addresses.

I created this tool to visualize route tables and diagnose network issues. You can copy and paste your route table here and the graphic will update. All the code is in the front-end, so if you add this blog to your home-screen then you can use the tool without internet access, which may be useful. It is currently in beta so if you see something incorrect then please submit a bug report and I will fix it ASAP.
